# 家族树应用部署指南

本指南将帮助您将家族树应用部署到互联网上，使其可以被其他朋友访问。

## 部署选项

### 1. 使用GitHub Pages

GitHub Pages是一个免费的静态网站托管服务，适合部署纯前端应用。

**步骤：**
1. 在GitHub上创建一个新的仓库
2. 将家族树应用的文件推送到仓库中
3. 在仓库设置中启用GitHub Pages，选择主分支作为源
4. 等待几分钟，您的应用将可以通过GitHub Pages提供的URL访问

### 2. 使用Vercel

Vercel是一个现代化的静态网站托管服务，提供免费的部署选项。

**步骤：**
1. 注册Vercel账号
2. 连接您的GitHub仓库或直接上传文件
3. Vercel会自动部署您的应用
4. 获得一个可访问的URL

### 3. 使用Netlify

Netlify是另一个流行的静态网站托管服务，提供免费的部署选项。

**步骤：**
1. 注册Netlify账号
2. 连接您的GitHub仓库或直接上传文件
3. Netlify会自动部署您的应用
4. 获得一个可访问的URL

### 4. 部署到自己的服务器

如果您有自己的服务器，可以将应用部署到服务器上。

**步骤：**
1. 将应用文件上传到服务器
2. 配置Web服务器（如Nginx或Apache）以提供静态文件服务
3. 确保服务器的防火墙允许HTTP/HTTPS流量
4. 您的应用将可以通过服务器的IP地址或域名访问

## 代码调整

由于这是一个纯前端应用，使用IndexedDB存储数据，您不需要对代码进行太多调整。但以下几点需要注意：

### 1. 确保所有资源路径都是相对路径

应用中的所有资源引用（如图像、CSS、JavaScript）都应该使用相对路径，以确保在不同的域名下都能正常工作。

### 2. 数据存储说明

- 应用使用IndexedDB存储数据，这是浏览器内置的数据库
- 数据存储在用户的浏览器中，不会被其他用户看到
- 每个用户的浏览器中都有自己独立的数据库
- 如果您希望在多用户之间共享数据，需要添加后端服务器和数据库

### 3. 跨域资源共享（CORS）

如果您的应用需要从其他域名加载资源（如图片），请确保这些资源的服务器配置了正确的CORS头部。

## 测试部署

部署完成后，请确保测试以下功能：

1. 应用能够正常加载
2. 能够添加和编辑家族成员
3. 家族树能够正确显示
4. 数据能够正常保存和加载
5. 不同浏览器中的兼容性

## 自定义域名

如果您希望使用自定义域名访问您的应用，可以：

1. 购买一个域名
2. 在域名注册商处配置DNS记录，将域名指向您的托管服务
3. 在托管服务中添加自定义域名

## 结论

家族树应用是一个纯前端应用，非常适合部署到静态网站托管服务上。由于它使用IndexedDB存储数据，每个用户都有自己的独立数据库，不需要后端服务器。

通过以上方法，您可以轻松地将家族树应用部署到互联网上，与朋友分享您的家族历史。
