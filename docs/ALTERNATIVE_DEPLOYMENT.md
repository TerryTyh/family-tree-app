# 替代部署方案

由于GitHub可能需要特殊网络环境才能访问，以下是一些替代的部署方案：

## 方案1：使用Gitee（码云）

Gitee是国内的代码托管平台，类似于GitHub，访问速度快，不需要特殊网络环境。

### 步骤：
1. 访问 https://gitee.com/ 注册账号（如果还没有）
2. 登录Gitee账号
3. 点击右上角的「+」按钮，选择「新建仓库」
4. 填写仓库信息：
   - 仓库名称：family-tree-app
   - 仓库介绍：家族树应用 - 记录家族历史与传承
   - 选择「公开」
   - 不要勾选「使用README文件初始化仓库」
5. 点击「创建仓库」按钮
6. 复制仓库的HTTPS地址
7. 在终端中运行以下命令（将 `<gitee-url>` 替换为复制的地址）：
   ```bash
   git remote add origin <gitee-url>
   git push -u origin main
   ```
8. 在Gitee仓库的「服务」选项卡中，启用「Gitee Pages」
9. 等待部署完成，您的应用将可以通过Gitee Pages提供的URL访问

## 方案2：使用Vercel的拖放部署

Vercel是一个现代化的静态网站托管服务，支持拖放部署，非常简单易用。

### 步骤：
1. 访问 https://vercel.com/ 注册账号（可以使用邮箱或GitHub账号）
2. 登录Vercel账号
3. 在仪表板页面，点击「New Project」按钮
4. 选择「Drag and Drop」选项
5. 找到您的家族树应用文件夹（/Users/tianyuehua/应用/family-tree-app）
6. 将整个文件夹拖放到Vercel的浏览器窗口中
7. Vercel会自动部署您的应用
8. 部署完成后，您将获得一个可访问的URL

## 方案3：使用Netlify的拖放部署

Netlify是另一个流行的静态网站托管服务，同样支持拖放部署。

### 步骤：
1. 访问 https://www.netlify.com/ 注册账号
2. 登录Netlify账号
3. 在仪表板页面，点击「New site from Git」按钮
4. 选择「Deploy manually」选项
5. 找到您的家族树应用文件夹（/Users/tianyuehua/应用/family-tree-app）
6. 将整个文件夹拖放到Netlify的浏览器窗口中
7. Netlify会自动部署您的应用
8. 部署完成后，您将获得一个可访问的URL

## 方案4：使用国内的静态网站托管服务

### 阿里云OSS + CDN
1. 注册阿里云账号
2. 开通对象存储OSS服务
3. 创建一个Bucket，设置为「公共读」
4. 上传家族树应用的所有文件到Bucket
5. 配置CDN加速（可选）
6. 通过OSS提供的访问URL访问您的应用

### 腾讯云COS
1. 注册腾讯云账号
2. 开通对象存储COS服务
3. 创建一个存储桶，设置为「公共读」
4. 上传家族树应用的所有文件到存储桶
5. 通过COS提供的访问URL访问您的应用

## 最简单的方案：压缩文件分享

如果您只是想与少数朋友分享，而不需要长期在线，可以：
1. 右键点击家族树应用文件夹
2. 选择「压缩」或「创建归档」
3. 将生成的压缩文件通过邮件、微信或其他方式发送给朋友
4. 朋友下载后解压，直接在浏览器中打开index.html文件即可

## 注意事项

无论使用哪种部署方案，请注意：
- 数据存储在用户的浏览器中，不会被其他用户看到
- 每个用户的浏览器中都有自己独立的数据库
- 如果您希望在多用户之间共享数据，需要添加后端服务器和数据库
